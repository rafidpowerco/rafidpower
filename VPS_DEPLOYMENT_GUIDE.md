# دليل النشر على خوادم الإنتاج (VPS Deployment Guide) 🚀

بعد شراءك لخادم VPS يعمل بنظام تشغيل Ubuntu 24.04 أو 22.04، اتبع هذه الخطوات البسيطة لتحويل المشروع بنقرة واحدة إلى بيئة إنتاج فعلية مفتوحة للعالم.

## 1. نقل الملفات إلى الخادم (Upload Files)

استخدم أي برنامج FTP (مثل FileZilla أو WinSCP) أو استخدم أوامر الـ `Github` لنقل مجلد المشروع بالكامل `rafid-scale-website` إلى المسار التالي في الـ VPS:
`/var/www/rafid-power`

## 2. تشغيل سكربت الإعداد التلقائي (Run Setup Script)

لقد قمت بإنشاء مجلد `vps_deployment_assets` يحتوي على كود تشغيل آلي. كل ما عليك فعله هو الدخول إلى الخادم (SSH) وتشغيل الأمر التالي:

```bash
cd /var/www/rafid-power
chmod +x vps_deployment_assets/vps_setup.sh
./vps_deployment_assets/vps_setup.sh
```

### ماذا سيفعل هذا السكربت؟

1. سيقوم بتحديث النظام بالكامل وتركيب جدار الحماية (Firewall).
2. سيقوم بتثبيت (Node.js) و (Python 3.11) و (Nginx).
3. سيقوم بربط وإعداد قاعدة بيانات المتجهات (ChromaDB) والمحرك الأساسي.
4. سيجهز الـ React كملفات بنية (Build).
5. سيربط قنوات البث المباشر (WebSockets) بشكل صحيح.
6. سيبدأ بتشغيل الخادم الخاص بالذكاء الاصطناعي (`web_ai_daemon.py`) ليبدأ بالعمل إلى الأبد في خلفية الخادم باستخدام (PM2).

## 3. وضع المفتاح السري (API Key)

بعد تنصيب المشروع، افتح المسار:
`/var/www/rafid-power/AI/.env`
وضع في داخله مفتاح `GEMINI_API_KEY` ومفاتيح التشفير المطلوبة ليعمل النظام الأساسي.

## 4. إعداد شهادة الأمان (أهم خطوة للـ WebSocket)

ميزة "لوحة الميزان المباشرة" لن تعمل في المتصفحات الحديثة إذا لم يكن الموقع مشفراً (HTTPS/WSS).
بعد ربط "الدومين" (`www.rafidpower.xyz`) بعنوان الـ `IP` الخاص بالخادم (من خلال لوحة تحكم الـ DNS الخاصة بك)، قم بتكوين الشهادة مجاناً بكتابة هذا الأمر:

```bash
sudo certbot --nginx -d rafidpower.xyz -d www.rafidpower.xyz
```

وأخيراً قم بإعادة تشغيل المنظومة:

```bash
sudo systemctl restart nginx
pm2 restart rafid-ai-daemon
```
