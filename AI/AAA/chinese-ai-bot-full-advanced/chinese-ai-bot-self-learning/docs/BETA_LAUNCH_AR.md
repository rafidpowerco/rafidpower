# إطلاق تجريبي احترافي (للمقربين)

هذا الدليل يجهّزك لإطلاق **نسخة تجريبية مغلقة** لاختبار الاستقرار والتجربة قبل الجمهور الأوسع.

## 1. ما الذي نعنيه بـ «تجريبي للمقربين»؟

- جمهور **محدود وموثوق** (عائلة، فريق صغير، شركاء).
- رابط أو دومين **لا يُعلن على وسائل التواصل** حتى تستقر الخدمة.
- هدف مزدوج: **التأكد من أن النظام يعمل** في بيئة حقيقية، وجمع **ملاحظات** وليس «إطلاقاً تسويقياً».

## 2. قائمة تحقق للمشغّل (قبل إرسال الدعوة)

### إلزامي تقريباً

| البند                          | ملاحظة                                                                                          |
| ------------------------------ | ----------------------------------------------------------------------------------------------- |
| `DATABASE_URL`                 | MySQL يعمل؛ نفّذ الترحيلات (`pnpm db:migrate` أو `RUN_MIGRATIONS_ON_START=1` في Docker).        |
| `JWT_SECRET`                   | سلسلة عشوائية طويلة؛ **لا تُشارك** ولا تُرفع لـ Git.                                            |
| OAuth                          | في لوحة مزوّد الهوية: **Redirect URI** يطابق عنوانك (`https://your-domain/api/oauth/callback`). |
| `LLM_BACKEND` + مفاتيح النموذج | تأكد أن `pnpm run verify:llm` (أو اختبار من لوحة التحكم) ينجح.                                  |

### يُنصح بشدة

| البند                     | ملاحظة                                                                                         |
| ------------------------- | ---------------------------------------------------------------------------------------------- |
| **HTTPS**                 | استخدم nginx أو Caddy أو خدمة استضافة توفر TLS؛ لا ترسل كوكي الجلسة على HTTP علني.             |
| `PUBLIC_APP_URL`          | يطابق الدومين العلني (للتذكير في السجلات والتشخيص).                                            |
| `INTERNAL_BOT_API_SECRET` | في الإنتاج: عيّن قيمة واستخدم الرأس `x-internal-bot-secret` لمسارات البوت التراثية إن احتجتها. |
| `TRUST_PROXY=1`           | إذا الخادم خلف وكيل عكسي يضيف `X-Forwarded-For`.                                               |
| `TELEGRAM_BOT_DISABLED=1` | إن لم تكن جاهزاً لتليجرام في التجربة.                                                          |

### واجهة «إصدار تجريبي»

- عند بناء الواجهة للإطلاق، عيّن **`VITE_SHOW_BETA_BANNER=1`** (في بيئة البناء أو ملف `.env` الذي يقرأه Vite) ليظهر شريط تنبيه احترافي أعلى التطبيق.
- المستخدم يمكنه إخفاء الشريط للجلسة فقط (يُخزَّن في `sessionStorage`).

```bash
# مثال Windows PowerShell قبل build
$env:VITE_SHOW_BETA_BANNER="1"; pnpm run build
```

## 3. نص جاهز لإرساله للمختبرين (يمكنك النسخ والتعديل)

```
السلام عليكم،

أطلقنا نسخة تجريبية داخلية من «منصة المساعد الذكي» ونرغب بملاحظاتكم.

الرابط: [ضع الرابط هنا]
تسجيل الدخول: عبر [OAuth / الطريقة التي فعّلتها]

نرجو:
1) تجربة تسجيل الدخول والمحادثة وصفحة «المعرفة» إن رغبتم.
2) الإبلاغ عن أي خطأ، صفحة بيضاء، أو بطء.
3) عدم مشاركة الرابط خارج مجموعتنا حتى ننتهي من التجربة.

شكراً لتعاونكم.
```

## 4. ماذا يختبرون؟ (قائمة قصيرة)

1. **تسجيل الدخول** والخروج.
2. **محادثة** جديدة + رسالة واحدة على الأقل.
3. (اختياري) **إضافة مقتطف معرفة** ثم سؤال يستدعي ذلك المقتطف.
4. **الإحصائيات / لوحة التحكم** — تحميل دون أخطاء واضحة.

## 5. جمع الملاحظات

- قناة واحدة واضحة: واتساب، بريد، أو نموذج بسيط.
- اطلب: **ماذا فعلت** → **ماذا توقعت** → **ماذا حدث** (ولقطة شاشة إن أمكن).

## 6. خصوصية وأمان في التجربة

- نبّه المختبرين: **لا يدخلوا أسراراً حقيقية** (كلمات مرور بنك، بيانات عملاء حساسة) في المحادثات التجريبية.
- راقب **سجلات الخادم** واستهلاك الـ LLM لتفادي تكاليف غير متوقعة.

## 7. بعد التجربة

- أغلق الوصول إن لزم (جدار نار، إيقاف الخدمة، أو إلغاء OAuth client).
- حدّث الإصدار في `package.json` عند الانتقال لمرحلة أوسع (مثلاً `1.0.0`).
- عطّل **`VITE_SHOW_BETA_BANNER`** عند الإطلاق الرسمي إن لم تعد تريد الشريط.
