# خارطة طريق: وكيل تجاري + مساعد لغوي (مفتوح المصدر + عملاء)

## ما الذي يمكن بناؤه واقعياً؟

- **مساعد لغوي تجاري**: نعم — عبر تجميع **مزودين** (APIs)، **نماذج محلية** (Ollama)، **قواعد بيانات العملاء**، و**سياسات اشتراك**.
- **«كل المصادر المفتوحة وكل الأكواد»**: لا يوجد منتج واحد «يجمع الإنترنت» قانونياً وتقنياً دون:
  - ترخيص صريح لكل مكون،
  - بنية فهرسة وامتثال (GDPR، حقوق نشر)،
  - تكلفة حوسبة هائلة.
- **التعلم الذاتي في منتج تجاري**: يعني عادة **تعلم من بيانات عملائك ضمن عقد** + **تحسين التوجيه والأدوات**، وليس «تدريب نموذج عالمي من كل GitHub».

## المبدأ التجاري (مستقبلك كمنتج)

1. **القيمة**: وكيل يفهم سياق العميل (مستنداته، سياساته، أدواته المعتمدة) + يحترم **الحدود والفوترة**.
2. **الامتثال**: شروط استخدام كل API، تراخيص المكتبات (MIT مقابل GPL)، وموافقة العميل على استخدام بياناته للتحسين.
3. **التوسع**: طبقات واضحة: **استيعاب → فهرسة → استرجاع → تنفيذ أدوات → تسجيل → فوترة**.

## مراحل مقترحة (حتى يصبح «تجاري بالكامل»)

| المرحلة | الهدف                                                         | مخرجات                      |
| ------- | ------------------------------------------------------------- | --------------------------- |
| **MVP** | مساعد لغوي + مسارات LLM (Forge / OpenRouter / مخصص) + حد معدل | لديك أساس في المشروع الحالي |
| **M1**  | تعدد مستأجرين (tenancy)، مفاتيح API لكل عميل، سجلات تدقيق     | جداول + سياسات              |
| **M2**  | RAG على مستندات العميل (S3/DB)، أدوات موقعة                   | فهرسة + بحث دلالي           |
| **M3**  | لوحة فوترة واستهلاك (tokens، طلبات، مقاعد)                    | تكامل Stripe أو محلي        |
| **M4**  | تحسين مستمر من **إشارات العميل** (تقييم، نجاح مهمة) فقط       | توسيع `learningEngine`      |

## جمع «أكبر عدد من الأكواد» — التحذير القانوني

- **لا تنسخ مستودعات بالجملة** دون ترخيص متوافق مع منتجك التجاري.
- **الاعتماد على الحزم من npm/pypi** مع الاحتفاظ بـ `LICENSE` و`NOTICE`.
- **الاستشارة القانونية** قبل بيع منتج يعتمد على تجميع كود طرف ثالث.

## ارتباط هذا المستودع

- النواة البرمجية المقترحة: `server/platform/commercialAgentKernel.ts` (نقاط توسيع، بدون ادعاءات وهمية).
- التشغيل والنموذج المخصص: `.env.example` و`LLM_BACKEND=custom` وDocker.
